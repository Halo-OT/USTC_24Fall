# 			稀疏一元多项式实验报告

## 1. 问题描述
本实验的目标是使用链表来实现 **稀疏一元多项式**。稀疏多项式是指大多数项的系数为零，仅存储系数不为零的项。主要功能是高效地插入、合并多项式项，并实现对一元稀疏多项式的几项基本操作。

- **输入并创建多项式**  (按指数升序或降序排列，系数浮点型，指数整型)
- **输出多项式**，项数+每项系数指数  (按指数升序或降序排列)
- **求和** :多项式相加
- **求差** :多项式相减
- **求值**
- **销毁**
- **清空**
- **修改**  (1插入新的结点、2删除已有结点、3修改已有结点的系数和指数)
- **微分**（N阶导数）
- **不定积分** 
- **定积分**

------



## 2. 算法描述
### (1) 数据结构描述
- **逻辑结构**：使用链表来表示多项式，每个节点存储多项式的一项，包括该项的系数和指数，按指数从大到小的顺序排列。并且将链表表头都存放在指针数组中。

- **存储结构**：
  
  - `Pitem` 结构体用于表示多项式中的一项，包括：
    - `xishu`：系数 （float）
    - `exp`：指数 （float ：考虑到 后面不定积分中说不定有 lnx 的输出 （把lnx标记为 -1.5次））
    - `next`：指向下一项的指针 
  
  - `Phead` 结构体用于表示多项式的头节点，包括：
  
    - `number`：多项式项的数量 （int）
    - `next`：指向多项式第一个节点的指针 
  
  - 用 `Phead* P[N] = { NULL }`：定义指针数组，指向各个多项式的头节点以达成如要求所示的存储结构：
  
    ><img src="/Users/halo/Library/Application Support/typora-user-images/image-20241022233022119.png" alt="image-20241022233022119" style="zoom:50%;" />
    >
    >
  
  **关键函数**：
  
  - `creatPhead`：初始化一个空的多项式（链表头）。
  - `creatPitem`：创建一个新项，包含给定的系数和指数。
  - `Pinsert`：将一项插入到多项式中，按指数递减顺序插入（可实现不按顺序插入），若已有相同指数的项则合并系数。

### (2) 程序结构
- **函数原型**：
  - `Phead* creatPhead()`：初始化多项式头节点。
  - `Pitem* creatPitem(float xishu, float exp)`：创建多项式的一项。
  - `Pinsert(Phead* head, float xishu, float exp)`：将一项插入到多项式中，若遇到相同指数的项则合并系数。
  - `Pmodify(Phead *head, float old_exp, float new_xishu, float new_exp)`:  修改目标项目指数、系数
  - `fixP(Phead *head)`:  修改链表 插入、删除、修改节点的系数&指数 
  - `creatP(Phead* head )`:  多项式建立函数（插入法建立）
  - `displayP(Phead *head)`:  多项式输出
  - `Pvalue(Phead *head, float x)`:  多项式求值函数
  - `clearP(Phead* head)`：多项式清空+free空间
  - `Phead* addP(Phead* head1, Phead* head2)`：多项式加法
  - `Phead* minusP(Phead* head1, Phead* head2)`：多项式减法
  - `Pderivative(Phead* head)`： 微分
  - `Pintegral(Phead *head)`：不定积分
  - `float Pdefinite_integrals(Phead* head, float x1, float x2)`：定积分

- **主函数**：

  - 在主函数中， 声明指针数组 `Phead* P[N] = { NULL }` 存放各个后续建立的链表的表头。

  - 用switch - case 在菜单界面选择不同的函数对多项式进行操作。

## 3. 调试与分析
- **测试数据**：
  测试使用了多种多项式组合，例如：
  
  - 多项式 1: \( 3x^5 + 2x^3 - 4x^2 + 6 \)
  - 多项式 2: \( 5x^4 - 3x^3 + x^2 - 9 \)
  - ……
  
- **遇到的问题 & 解决方法**：
  
  1. **内存泄漏**：在合并相同指数的项时，未释放多余节点，导致内存泄漏问题。为此，添加了 `free(newitem)` 来释放不需要的节点。

  2. **边界情况**：
  
     - `Pinsert`插入系数为 0 的项时，必须跳过这些项，防止它们进入链表。
     - `displayP`、`Pintegral`：多项式输出时，若要输出不定积分的结果，可能会遇到 lnx 的输出，此时通过在 `Pintegral` 中把 x^-1 项的积分结果的指数储存为 -1.5 加以区分和针对性输出。
  
     - `Pderivative`：求微分
       - 根据助教哥哥的要求， 在微分结束时， 应输出 0 而不是 “多项式为空”，由此更改输出逻辑为，每一轮微分后，判断一次多项式是否为 0 ，为 0 则直接输出，跳过`displayP`（它会在多项式没有内容（每一项均为 0 ）时输出 ”多项式为空喵～“。
       - 对于输入的过大的求导次数， 函数会针对性输出，“该多项式在第%d次求微分前时已为 0 喵～ 无法继续求微分喵～”， 由此保证了程序的健壮性。
  
  3. **重复指数项的插入：**
  
     - 问题：当插入与现有项指数相同的项时，程序应将系数合并。在调试初期，因索引处理不当，合并出现错误。
     - 解决方法：修正查找相同指数项的逻辑，在合并前正确定位到相同指数的位置。

## 4. 时空复杂度分析
- **时间复杂度：**
  - **插入操作：** 插入操作需要遍历链表，找到适合插入的位置。在最坏情况下，需要遍历所有项，因此时间复杂度为 O(n)，其中 n 是多项式项的数量。
  - **删除操作：** 删除操作同样需要遍历链表，因此时间复杂度也是 O(n)。
  - **修改操作：** 修改操作包括删除旧项和插入新项，因此整体时间复杂度为 O(n)。
  - **求导和积分：** 这些操作需要遍历多项式的所有项，时间复杂度为 O(n)。

- **空间复杂度：**
  - 空间复杂度为 O(n)，其中 n 是多项式的项数，因为每一项都存储在链表的一个节点中。

## 6. 实验体会和收获

通过本次实验，我加深了对使用链表处理稀疏多项式的理解。实现插入、删除和修改操作时，需要特别注意指针的操作和内存管理。此外，在求导和积分操作时，程序需要对多项式项的次幂和系数进行准确的操作和处理。

本次实验的主要收获有：

- 重温了链表知识。
- 通过调试，学习了如何有效避免和解决内存泄漏问题。
- 认识到在实现算法时，处理边界情况和极端输入的重要性。
